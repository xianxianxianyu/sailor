# Sailor v2 完整架构方案

## 1. 产品定位

Sailor 是一个**个人信息聚合 + 智能分类 + 知识管理**平台。

核心用户故事：
> 我订阅了很多 RSS 源和非 RSS 信息源，我希望一键抓取所有源的最新内容，
> 系统根据我的偏好自动打标分类，像 GitHub Trending 一样按标签分组展示，
> 我可以点击标题跳转原文，把感兴趣的文章收藏到知识库，
> 知识库的收藏行为会反哺回我的偏好，让下次打标更精准。

核心流程：**订阅源管理 → 一键抓取 → LLM 智能打标 → Trending 报告 → 收藏到知识库 → 偏好反哺**

## 12. 技术决策

1. **侧边导航而非标签页**：侧边导航占用空间小（64px），主内容区最大化，且导航项固定可见，用户随时知道自己在哪。

2. **页面级组件而非全塞一屏**：每个功能独立成页，避免信息过载。Trending 是默认页，因为它是最高频操作。

3. **LLM 替代硬编码 TOPIC_KEYWORDS**：原来只能匹配 5 个固定主题，TaggingAgent 通过 LLM 动态识别，且优先复用用户已有标签。

4. **Weight 驱动排序**：tag weight 同时影响标签云大小、Trending 分组排序、LLM 打标优先级，形成统一的偏好信号。

5. **Feed 管理前置**：订阅源是数据入口，用户需要能方便地管理。支持 OPML 批量导入 + 手动添加 + 启用/暂停/删除。

6. **Pipeline 原子性**：`/trending/pipeline` 将抓取+打标+生成合为一个 API，前端一键触发。

7. **移除硬编码 KB**：`ensure_defaults()` 不再创建默认 KB，用户完全自主管理。
